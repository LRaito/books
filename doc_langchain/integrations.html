组件集成

LangChain 提供了一个广泛的生态系统，包含 1000 多个集成，涵盖聊天和嵌入模型、工具和工具包、文档加载器、向量存储等等。 Provider 是指 LangChain 与之集成的第三方服务或平台，用于访问人工智能功能，例如聊天模型、嵌入模型和向量存储。这些提供商拥有独立的 langchain-provider 软件包，以改进版本控制、依赖关系管理和测试。

检索器

检索器是一个接口，通过非结构化查询返回文档，比向量存储更通用。检索器并不存储文档，仅仅返回文档。检索器可以通过向量存储构建，但是也可以直接从 Wiki 搜索返回。 检索器接收文本输入，返回一些文档清单。 所有向量存储可以转换为检索器。

文本分割器

文本分割器将大型文档分割成更小的块，这些块可以单独检索，并且适合模型上下文窗口限制。 文档拆分有多种策略，每种策略都有其自身的优势。

对于大多数使用场景，建议首先使用默认策略RecursiveCharacterTextSplitter。它在保持上下文完整性和管理数据块大小之间取得了良好的平衡。这种默认策略开箱即用，效果显著，只有当需要针对特定​​应用程序进行性能微调时才应考虑调整。

Text structure-based 基于文本结构

文本自然而然地组织成段落、句子和单词等层级单元。我们可以利用这种固有的结构来指导我们的分割策略，创建能够保持自然语言流畅性、维持内部语义一致性并适应不同文本粒度级别的分割结果。 LangChain 的 RecursiveCharacterTextSplitter 实现了这些概念：

  • 尝试保持较大单元（例如段落）的完整性
  • 如果一个单元超过块大小，它将移到下一级（例如，句子）。
  • 如有必要，此过程会一直向下进行到单词级别。 
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(chunk_size=100, chunk_overlap=0)
texts = text_splitter.split_text(document)

Length-based 基于长度

一种直观的策略是根据文档长度进行拆分。这种简单而有效的方法可以确保每个文件块的大小不超过指定限制。基于长度拆分文档的主要优点：

  • 简单易行的实施
  • 一致的块大小
  • 可轻松适应不同模型的要求

基于长度的分割类型：

  • Token-based：基于 token
  • Character-based：基于 character 
from langchain_text_splitters import CharacterTextSplitter

text_splitter = CharacterTextSplitter.from_tiktoken_encoder(
    encoding_name="cl100k_base", chunk_size=100, chunk_overlap=0
)
texts = text_splitter.split_text(document)

Document structure-based 基于文档结构

有些文档具有固有的结构，例如 HTML、Markdown 或 JSON 文件。在这种情况下，根据文档结构进行拆分会很有帮助，因为结构通常会将语义相关的文本自然地分组。基于结构的拆分的主要优势：

  • 保持文档的逻辑结构
  • 在每个数据块内保持上下文
  • 对于检索或摘要等下游任务，可能更有效。

基于结构的分割示例：

  • Markdown ：根据标题拆分（例如#，，，##）###
  • HTML：使用标签进行分割
  • JSON：按对象或数组元素拆分
  • 代码：按函数、类或逻辑块拆分

嵌入模型

嵌入模型将原始文本（例如句子、段落或推文）转换为固定长度的数字向量，该向量包含语义含义（semantic meaning）。这些向量使机器能够基于含义而非确切的词语来比较和搜索文本。 实际上，这意味着在向量空间中，具有相似概念的文本会被放置在彼此靠近的位置。例如，嵌入向量不仅可以匹配"机器学习"这个短语，还可以识别出讨论相关概念的文档，即使它们使用的措辞有所不同。

工作原理

  • 向量化：该模型将每个输入字符串编码为一个高维向量。
  • 相似度评分：使用数学指标比较向量，以衡量底层文本的关联程度。

相似性指标

比较嵌入向量时通常使用以下几种指标：

  • 余弦相似度：衡量两个向量之间的角度。
  • 欧氏距离：测量点之间的直线距离。
  • 点积：衡量一个向量在另一个向量上的投影程度。 
import numpy as np

def cosine_similarity(vec1, vec2):
    dot = np.dot(vec1, vec2)
    return dot / (np.linalg.norm(vec1) * np.linalg.norm(vec2))

similarity = cosine_similarity(query_embedding, document_embedding)
print("Cosine Similarity:", similarity)

LangChain 为矢量存储提供了一个统一的接口，使您能够：

  • add_documents：将文档添加到存储。
  • delete：按ID删除已存储的文档。
  • similarity_search：查询语义相似的文档。

文档加载器

文档加载器提供了一个标准接口，用于将来自不同来源（例如 Slack、Notion 或 Google Drive）的数据读取为 LangChain 的文档格式。这确保了无论数据来源如何，都能以一致的方式处理数据。 所有文档加载器都实现了该BaseLoader接口。

接口

每个文档加载器都可以定义自己的参数，但它们共享一个通用的 API：

  • load()：一次性加载所有文档。
  • lazy_load()：延迟加载文档流，适用于大型数据集。

示例

from langchain_community.document_loaders.csv_loader import CSVLoader

loader = CSVLoader(
    ...  # Integration-specific parameters here
)

# Load all documents
documents = loader.load()

# For large datasets, lazily load documents
for document in loader.lazy_load():
    print(document)

键值存储

LangChain 提供了一个键值存储接口，用于按键存储和检索数据。LangChain 中的键值存储接口主要用于缓存嵌入内容。

接口

所有产品均BaseStores支持以下接口

接口方法 说明
mget(key: Sequence[str]) -> List[Optional[bytes]] 获取多个键的内容，如果键不存在则返回 None
mset(key_value_pairs: Sequence[Tuple[str, bytes]]) -> None 设置多个键的内容
mdelete(key: Sequence[str]) -> None 删除多个键
yield_keys(prefix: Optional[str] = None) -> Iterator[str] 返回存储中的所有键，可以选择性地按前缀进行过滤
